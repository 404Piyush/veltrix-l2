# Veltrix: Second Build in Public Post

## LinkedIn Post

Veltrix update #2: moved from local devnet to a Sepolia-backed OP Stack testnet.

This milestone was mostly infra-heavy: not just "start the containers", but getting the real rollup pipeline to behave correctly against Sepolia.

What is now working:
- Sepolia-backed op-geth, op-node, op-batcher, and op-proposer are running.
- L1 derivation caught up from a large backlog; l1_cursor_lag is now around 4 blocks.
- safe_l2 and finalized_l2 are advancing from real L1 batch data.
- op-proposer moved past the stale duplicate proposal issue and is now publishing newer output roots successfully.
- Prometheus and Grafana are live for local + Sepolia stack monitoring.
- Health checks now show unsafe, safe, finalized, unsafe-safe lag, and L1 cursor lag for both environments.

The interesting part: unsafe blocks were easy to produce, but getting safe/finalized movement right required waiting for OP Stack derivation to replay Sepolia L1 batch data in order. That was a good reminder that an L2 is only meaningful when the L1 data path is correct.

Next steps:
- Bridge UI
- Faucet
- Public explorer polish
- VPS/domain deployment once the technical path is stable

This is still a testnet milestone, not a public launch yet, but the core rollup infra is now much closer to production-style operation.

## Do Not Screenshot

- `.env`
- private keys
- seed phrases
- full RPC URLs with provider tokens
- wallet key files
- any terminal command that prints secrets
